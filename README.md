# 12306 Smart Query for OpenClaw

　　一个专为 OpenClaw 打造的智能 12306 火车票查询技能，支持直达优先、指定中转、多站换乘以及智能多跳自动规划，特别优化了普速列车（ZTK）的复杂路线探索，并内置本地图缓存机制，让重复查询秒级响应。

---

## ✨ 功能亮点

- **直达优先**：自动查询直达车票，无直达时智能降级为中转方案。
- **指定途经站筛选**：支持强制中转，并精准筛选出经过特定城市的换乘方案。
- **多站顺序换乘**：可指定多个必经站点，自动排列组合并验证每一段车票。
- **智能多跳自动规划**：无需指定具体途经点，基于图搜索算法自动发现最优换乘路线（尤其适合“只坐慢车”的长途旅行）。
- **列车类型过滤**：支持组合筛选（如 `ZTK`​、`GD`​、`G`​ 等），满足不同速度需求。
- **本地缓存图**：历史查询的车次信息会自动构建成图，后续查询优先走缓存，大幅提升速度。
- **同车换乘过滤**：自动剔除 12306 官方接口返回的无效“同车换乘”冗余数据。

## 🔧 安装与配置

### 1. 克隆或下载本技能

　　将整个技能文件夹放入 OpenClaw 的 skills 目录（例如 `~/.openclaw/skills/12306-smart-query`​）。

### 2. 安装 Python 依赖

```bash
git clone https://github.com/52Herts-ux/12306-smart-query.git
cd 12306-smart-query
pip install -r requirements.txt
```

### 3. 启动 12306-mcp 服务

　　该技能依赖 `12306-mcp`​ 提供底层数据。推荐使用 `npx`​ 一键启动（确保已安装 Node.js）：

```bash
# 默认监听 127.0.0.1:8080，仅限本地访问（安全）
npx -y 12306-mcp --host localhost --port 8080
```
　　若需长期后台运行，可配合 `pm2`​ 或 `screen`​。
  

### 4. 修改技能配置

　　打开 `scripts/train_query.py`​，确认 `MCP_URL`​ 指向正确的服务地址（默认为 `http://127.0.0.1:8080/sse`​）。若服务运行在其他端口或主机，请相应修改。

## 📖 使用说明

　　技能通过命令行脚本 `train_query.py`​ 对外提供服务，OpenClaw 会根据用户意图自动拼接参数并调用。

### 命令格式

```bash
python scripts/train_query.py <出发地> <目的地> [选项]
```

### 参数详解

|参数|说明|示例|
| ------| --------------------------------------------------| ------|
|​`--date YYYY-MM-DD`​|指定查询日期，默认当天|​`--date 2026-04-15`​|
|​`--via <城市/车站>`​|强制中转并筛选途经该城市的方案（仅两段式）|​`--via 郑州`​|
|​`--train-type <类型串>`​|限定列车类型：G/D/Z/T/K/O/F/S，可组合|​`--train-type ZTK`​|
|​`--multi <城市列表>`​|指定多个必经站点，用逗号分隔，脚本会尝试所有排列|​`--multi 郑州,石家庄`​|
|​`--auto-plan`​|**智能多跳规划**：自动探索多段换乘路线|​`--auto-plan`​|
|​`--max-hops N`​|配合 `--auto-plan`​，限制最大换乘次数（默认4）|​`--max-hops 5`​|
|​`--refresh`​|强制刷新，忽略本地缓存重新查询|​`--refresh`​|

### 查询模式示例

#### ① 直达优先（默认）

```bash
python scripts/train_query.py 武汉 北京 --date 2026-04-15
```

#### ② 单中转筛选

```bash
python scripts/train_query.py 武汉 北京 --via 郑州 --train-type G
```

#### ③ 多站顺序换乘

```bash
python scripts/train_query.py 新乡 昆明 --multi 郑州,怀化 --train-type ZTK
```

#### ④ 智能多跳自动规划（推荐长途慢车）

```bash
python scripts/train_query.py 新乡 昆明 --auto-plan --train-type ZTK --max-hops 4
```

## 💾 缓存机制

- 所有查询到的车次信息（包括直达和中转分段）都会自动存入 `train_cache_graph.json`​。
- 缓存以**图结构**存储，车站为节点，车次为边。
- 再次查询时，优先在缓存图中搜索路径，命中后几乎零延迟返回。
- 缓存有效期默认为 **2 天**（可在脚本中调整 `CACHE_EXPIRE_DAYS`​）。
- 使用 `--refresh`​ 参数可跳过缓存，强制实时查询并更新缓存图。

> 随着使用次数的增加，缓存图会越来越丰富，查询效率也会逐步提升。

## 🔒 安全建议

- **12306-mcp 服务请务必绑定** **​`localhost`​**​：启动时使用 `--host localhost`​（默认即为此），避免端口暴露到公网。
- **配合云防火墙**：在云服务器安全组中确保 8080 端口不对公网开放。
- **仅限本地调用**：OpenClaw 与 12306-mcp 运行在同一台机器，通过 `127.0.0.1`​ 通信即可。

## ❓ 常见问题

### Q: 为什么自动规划返回 0 条路线？

　　A: 可能是当天确实无符合条件的车次组合。尝试放宽列车类型（如 `--train-type GZTK`​）、增加最大换乘次数（`--max-hops 5`​）或更换日期。

### Q: 查询速度很慢怎么办？

　　A: 首次查询需调用 12306 接口，耗时较长。后续相同或相关查询会直接走本地缓存，速度极快。若需立即加速，可手动扩展 `ZTK_HUBS`​ 字典加入更多枢纽站。

### Q: 如何清空缓存重新开始？

　　A: 直接删除技能目录下的 `train_cache_graph.json`​ 文件即可。
