# 12306 智能火车票查询技能

## 技能名称

　　​`12306-smart-query`​

## 描述

　　这是一个强大的 12306 火车票查询技能，支持**直达优先**、**指定途经站中转**、**多站顺序换乘**以及**智能多跳自动规划**。特别优化了普速列车（ZTK）的复杂换乘路线探索，并内置**本地图缓存**机制，大幅提升重复查询的响应速度。

## 触发器

　　当用户的输入包含以下意图或关键词时，推荐激活此技能：

- 查询火车票、车次、余票、票价
- “从 A 到 B 的火车”
- “中转”、“换乘”、“经过 C”、“途经”
- “只坐慢车”、“只要 Z/T/K 车次”
- “买长坐短”、“多停几站”
- “帮我规划一条从 A 到 B 的火车路线”

## 所需工具

- Python 3.8+
- 依赖库：`mcp`​, `httpx`​, `anyio`​（已在 `requirements.txt`​ 中声明）
- 运行中的 `12306-mcp`​ 服务（MCP 协议服务端）

## 安装与配置

1. 将技能目录放置于 OpenClaw 的 skills 文件夹下。
2. 安装 Python 依赖：  

    ```bash
    pip install -r requirements.txt
    ```
3. 通过执行：`npx -y 12306-mcp -host localhost --port 8080` 启动 MCP 服务端。
3. 确保 `12306-mcp`​ 服务已启动，并修改 `train_query.py`​ 中的 `MCP_URL`​ 为实际地址（默认为 `http://localhost:8080/sse`​）。
4. 可选：若服务部署在其他 IP/端口，可通过环境变量 `MCP_SERVER_URL`​ 覆盖。

## 使用说明

### 命令格式

```bash
python scripts/train_query.py <出发地> <目的地> [选项]
```

### 选项参数

|参数|说明|示例|
| ------| --------------------------------------------------------------------------| ------|
|​`--date YYYY-MM-DD`​|指定查询日期，默认为当天|​`--date 2026-04-15`​|
|​`--via <城市/车站>`​|强制中转并筛选途经该城市的方案（单中转）|​`--via 郑州`​|
|​`--train-type <类型串>`​|限定列车类型，如 `G`​(高铁) `D`​(动车) `Z`​(直达) `T`​(特快) `K`​(快速) `F`(复兴号) `S`(智能动车)等，可组合|​`--train-type ZTK`​|
|​`--multi <城市列表>`​|指定多个必经站点，用逗号分隔，脚本会尝试所有排列并返回可行的顺序换乘方案|​`--multi 郑州,石家庄`​|
|​`--auto-plan`​|**智能多跳规划**：不指定具体途经站，自动探索符合条件的多段换乘路线|​`--auto-plan`​|
|​`--max-hops N`​|与 `--auto-plan`​ 配合，设置最大换乘次数（默认4）|​`--max-hops 5`​|
|​`--refresh`​|强制刷新，忽略本地缓存，重新调用12306接口|​`--refresh`​|

### 查询模式详解

#### 1. 直达优先（默认）

　　未指定 `--via`​ 或 `--auto-plan`​ 时，脚本优先查询直达车票。若无直达则自动降级为官方推荐的中转方案（两段式），并自动过滤“同车换乘”的无效结果。

```bash
python scripts/train_query.py 武汉 北京 --date 2026-04-15
```

#### 2. 单中转筛选

　　指定 `--via`​ 时，脚本直接调用中转接口，并筛选出中转站包含指定城市的方案，同时支持列车类型过滤。

```bash
python scripts/train_query.py 武汉 北京 --via 郑州 --train-type G
```

#### 3. 多站顺序换乘

　　通过 `--multi`​ 指定多个必经点，脚本会枚举所有可能顺序，并验证每一段是否有可行车次，返回完整联程方案。

```bash
python scripts/train_query.py 新乡 昆明 --multi 郑州,怀化 --train-type ZTK
```

#### 4. 智能多跳自动规划

　　**核心功能**：当用户只说“从新乡到昆明，只坐慢车”时，使用 `--auto-plan`​ 启动图搜索算法，基于本地缓存图 + 实时枢纽探索，自动发现一条或多条符合要求的换乘路线（如新乡→郑州→武昌→长沙→贵阳→昆明）。

```bash
python scripts/train_query.py 新乡 昆明 --auto-plan --train-type ZTK --max-hops 4
```

### 缓存机制

- 查询结果（车次边信息）会自动保存到本地 `train_cache_graph.json`​ 文件中。
- 再次查询相同或相关路线时，优先从缓存图搜索路径，**秒级返回**。
- 缓存有效期可通过 `CACHE_EXPIRE_DAYS`​ 变量调整（默认2天）。
- 使用 `--refresh`​ 可跳过缓存，强制实时查询并更新缓存。

## 返回结果格式

　　脚本输出为 JSON 对象，包含以下通用字段：

```json
{
  "query_type": "auto_plan",          // 查询类型：direct / interline_with_via / multi_hop / auto_plan
  "date": "2026-04-15",
  "from": "新乡",
  "to": "昆明",
  "train_types": "ZTK",
  "total_routes": 2,
  "routes": [ ... ],                  // 路线数组，每条路线由多个 legs 组成
  "cached": true                      // 是否来自缓存（仅 auto_plan 模式包含）
}
```

　　具体字段根据查询类型略有差异，详见脚本输出。

## 技能开发与调试

- 修改 `MCP_URL`​ 或增加枢纽站可编辑脚本开头的常量。
- 缓存文件 `train_cache_graph.json`​ 可手动删除以清空缓存。
- 运行脚本时会向 stderr 输出状态信息（如缓存命中提示），不影响 JSON 结果解析。

## 示例对话

　　**用户**：帮我查一下后天从新乡到昆明的火车票，不想坐高铁，最好能便宜点。

　　**助手（内部调用）** ：

```bash
python scripts/train_query.py 新乡 昆明 --auto-plan --train-type ZTK --date 2026-04-12
```

　　**返回**：显示 3 条换乘路线，其中一条为“新乡→郑州→武昌→怀化→昆明”，并列出各段车次、时间及票价。

　　**用户**：这路线不错，但我还想在襄阳停一下。

　　**助手（内部调用）** ：

```bash
python scripts/train_query.py 新乡 昆明 --multi 襄阳 --train-type ZTK --date 2026-04-12
```

　　**返回**：返回所有必须经过襄阳的换乘方案（如新乡→襄阳→怀化→昆明）。

---

　　**注意事项**：

- 确保 `12306-mcp`​ 服务在运行且网络通畅。
- 多跳自动规划在首次查询时可能耗时较长（取决于枢纽站数量和网络延迟），后续缓存命中后极快。
- 若查询无结果，可尝试放宽列车类型、增加最大换乘次数或更换日期。
